Little Navmap Features and Limitations

Version 0.9.8.develop

Little Navmap Manual

Features

General

  • Supported platforms: Windows XP/7/8/10, Linux, Mac OS X.
  • Conservative user interface without experiments: Using a main menu, context menus, toolbars, dock windows, undo, recent file lists and more.
  • Written using C++ and Qt - It is fast.
  • The program saves almost all settings, like flight plan, windows positions, searches and more and restores everything on start up.
  • Most question or information dialogs can be disabled.
  • Support for Flight Simulator X, Flight Simulator - Steam Edition, Prepar3D v2 and Prepar3D v3 (partially untested).
  • Data is collected from flight simulator stock and add-on files. No need to buy or download external data like AIRAC cycles.
  • Weather display from ActiveSkyNext, NOAA or Vatsim in tooltips and information panels.
  • Scenery library databases for flight simulators are stored separately and can be changed on the fly.
  • Little Navmap can be run on computers that do not contain any simulator installation at all. Just copy the scenery library database from your flight simulator computer.
  • Airport and navaid icons are used throughout the whole program for easier recognition.
  • Information panels show all about airports or navaids.
  • Exhaustive search function for airports and navaids that allows filtering by an unimaginable amount of criteria.
  • Distance based search for airports or navaids around a center point. Can be combined with all other search options.
  • Mac OS X and Linux versions in preparation.

Flight Plan and Routing

  • Can read and write FSX PLN format (XML).
  • Drag and drop flight plan editing on the map.
  • Easy flight plan editing in the table view, like moving and deleting waypoints.
  • Can merge flight plans or flight plan fragments.
  • Automatic routing (does not require online services) for high/low altitude airways, VOR/NDB and preset cruise altitude.
  • Routing is based entirely on flight simulator data. No third party data needed and not online services used.
  • Elevation profile display for flight plan. Shows minimum altitude for flight plan.
  • Undo/redo for all flight plan edits.
  • Robust against malformed flight plans. Does not put unknown waypoints far off the route.

Map

  • Uses two free online maps (OpenStreetMap and OpenTopoMap) as well as two simple offline maps. The offline maps are included.
  • No need to download and install any additional elevation or map data.
  • Multiple map themes including hill shading and two map projections.
  • Airport display iconography based on real world VFR maps thus combining plenty of information in a small symbol and a few lines of text.
  • Detailed airport diagrams displayed in the map (not a separate window) including parking, taxiways, displaced thresholds, overrun areas and much more.
  • Add-on airport names are shown in italic text for easier recognition.
  • Distance measurement lines and configurable range rings.
  • Can load and display Google Earth KML files.
  • History of map position and zoom distance like a web browser.
  • Map display configuration including detail level on the toolbar.
  • Symbol and text sizes configurable in options dialog.
  • Detailed tooltips on map display

Simulator aircraft

  • Shows simulator user aircraft on the map including flight plan progress, ambient parameters like wind, temperature, pressure and more (similar to a flight management computer). Needs Little Navconnect which is free too.
  • Shows track of simulator user aircraft.
  • Using Little Navconnect as an agent between the simulator and Little Navmap saves the pain of remote SimConnect setup.

Limitations

Current limitations which will most likely be solved in future updates.

  • No display and flight plan integration for approach procedures and transitions.
  • No support for AIRAC map data.
  • Little Navmap does not run on flight simulator screen full screen mode.
  • No zooming and scrolling in flight plan elevation profile view.
  • No aircraft performance profiles (climb speed, cruise speed, fuel flow, etc.).
  • User defined waypoints not supported.
  • Support for other weather programs than Active Sky Next is missing.
  • Weather stations should be displayed of on the map.
  • North Atlantic and Pacific tracks are not used.
